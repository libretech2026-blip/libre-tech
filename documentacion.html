1. Descripción general del proyecto

LIBRE TECH es una tienda en línea de productos de tecnología enfocada en el mercado colombiano. Permite a los usuarios navegar productos, filtrarlos por categoría y marca, agregarlos a un carrito de compras y finalizar pedidos a través de WhatsApp. El sistema incluye un panel de administración para gestionar el catálogo de productos.

Características principales

  • Catálogo de 66+ productos con navegación por categoría y marca
  • Búsqueda en tiempo real con resultados desplegables (desktop y mobile)
  • Carrito de compras y lista de deseos con persistencia
  • Página de detalle de producto con galería, zoom, colores, specs y reseñas
  • Finalización de pedidos vía WhatsApp o proceso de checkout
  • Autenticación con Supabase (email/contraseña + Google OAuth)
  • Panel de administración con 10 módulos (productos, pedidos, banners, PQRs, reseñas, usuarios, etc.)
  • Sistema de banners visual con minimapa interactivo y drag & drop
  • Sistema de PQRs (Peticiones, Quejas, Reclamos, Sugerencias)
  • Reseñas y calificaciones por producto con moderación admin
  • Importación masiva de productos desde archivos CSV/TXT
  • Base de datos PostgreSQL en Supabase con RLS (Row Level Security)
  • Diseño responsive, tema oscuro/claro y UI moderna

4. Módulos del sistema

4.1 Store (app.js)

Módulo IIFE que maneja la lógica principal de la tienda.

  • renderProducts() — Renderiza las tarjetas de producto en el grid con filtros
  • renderCategories() — Genera chips de filtro por categoría dinámicamente
  • renderBrandSubFilters() — Muestra sub-filtros de marca al seleccionar categoría
  • renderSearchDropdown() — Búsqueda en tiempo real con dropdown de resultados
  • initHeroCarousel() — Carrusel hero con banners del admin o productos destacados
  • renderPromoBanners() — Banners laterales configurables desde admin (izq/der)
  • renderPromoPhotoBanners() — Banners full-width configurables desde admin
  • renderSocialLinks() — Redes sociales del footer desde admin
  • Wishlist — Lista de deseos con sidebar y persistencia

4.2 Cart (cart.js)

Módulo IIFE para el carrito de compras.

  • addItem(productId) — Agrega un producto al carrito (o incrementa cantidad)
  • removeItem(productId) — Elimina un producto del carrito
  • updateQuantity(productId, delta) — Ajusta la cantidad (+/-)
  • sendToWhatsApp() — Genera mensaje formateado y abre WhatsApp
  • checkout() — Proceso de pago con formulario (nombre, email, teléfono)
  • showToast(message, type) — Notificaciones toast con auto-eliminación
  • formatPrice(price) — Formato COP (pesos colombianos)

4.3 Auth (auth.js)

Módulo IIFE de autenticación con Supabase.

  • Supabase Auth — Login con email/contraseña y registro de cuentas
  • Google OAuth — Autenticación OAuth 2.0 con Google vía Supabase
  • onAuthChange() — Escucha cambios de sesión y actualiza la UI
  • renderOrders() — Historial de pedidos del usuario desde Supabase
  • Panel dropdown con estados autenticado/no autenticado

4.4 Supabase Client (supabase-client.js)

Wrapper centralizado para todas las operaciones con Supabase.

  • Productos — syncProducts, getCachedProducts, insertProduct, updateProduct, deleteProduct, pushAllProducts
  • Pedidos — saveOrder, getOrders, getAllOrders, updateOrderStatus
  • PQRs — submitPqr, getUserPqrs, getAllPqrs, replyPqr
  • Reseñas — getProductReviews, submitReview, deleteReview, getAllReviews
  • Usuarios — listUsers, deleteUser, updateUserPassword
  • Storage — uploadImage (Supabase Storage para imágenes)
  • Auth — getSession, getUser, isAdmin, onAuthChange

4.5 Product Detail (product-detail.js)

Módulo para la página de detalle de producto.

  • Galería de imágenes — Imagen principal con zoom (scroll + pinch)
  • Reseñas y calificaciones — Formulario de reseña y listado desde Supabase
  • Colores y especificaciones — Selector de colores y tabla de specs
  • Productos relacionados — Sugerencias por categoría

6. Funcionalidades detalladas

6.1 Búsqueda en tiempo real

Al escribir en la barra de búsqueda, se ejecuta una búsqueda con debounce (300ms) que filtra los productos activos por nombre, descripción, categoría y marca. Los resultados aparecen en un dropdown. En mobile, hay un botón dedicado que abre un overlay de búsqueda a pantalla completa.

6.2 Sistema de filtros (Categorías + Marcas)

Los filtros funcionan en dos niveles:

  • Nivel 1 — Categorías: Chips generados dinámicamente desde los productos activos.
  • Nivel 2 — Marcas: Sub-filtros que aparecen al seleccionar una categoría.

6.3 Autenticación con Supabase

La autenticación se maneja a través de Supabase Auth. Soporta registro e inicio de sesión con email/contraseña, y también Google OAuth 2.0. La sesión se gestiona automáticamente con onAuthStateChange(). El panel dropdown muestra formulario de login cuando no hay sesión, y nombre + historial de pedidos + enlace PQR cuando está autenticado.

6.4 Página de detalle de producto

Cada producto tiene su propia página (producto.html?id=UUID) con:

  • Galería de imágenes con zoom (scroll en desktop, pinch en mobile)
  • Selector de colores disponibles
  • Tabla de especificaciones técnicas
  • Sistema de reseñas con calificación de 1-5 estrellas
  • Formulario para dejar reseña (usuarios autenticados)
  • Productos relacionados por categoría

6.5 Sistema de banners (Admin → Visualización)

El admin tiene un minimapa interactivo de la página que muestra 6 posiciones de banner. Al hacer clic en una posición se abre el formulario para crear/editar el banner de esa zona. La tabla de banners soporta:

  • 6 posiciones: Hero Carrusel, Lateral Izquierdo, Después de Destacados, Lateral Derecho, Después de Categorías, Antes del Footer
  • Drag & drop: Arrastra filas para reordenar banners
  • Botones ↑↓: Subir/bajar banners con un clic
  • Sincronización: Los cambios en admin se reflejan automáticamente en la tienda (via localStorage)
  • Producto vinculado: Cada banner puede enlazar a un producto específico
  • Fechas: Programar banners con fecha de inicio y fin

6.6 Sistema de PQRs

Los usuarios autenticados pueden enviar Peticiones, Quejas, Reclamos y Sugerencias desde pqr.html. Los PQRs se almacenan en Supabase con RLS (cada usuario solo ve los suyos, admin ve todos). El admin puede responder PQRs desde el panel.

6.7 Reseñas y calificaciones

Los usuarios autenticados pueden calificar productos (1-5 estrellas) y dejar comentarios. Las reseñas se almacenan en Supabase (tabla reviews). El admin puede moderar (eliminar reseñas inapropiadas) desde la pestaña Reseñas del panel.

6.8 Base de datos Supabase

El proyecto usa Supabase como backend con las siguientes tablas:

TablaDescripciónRLS
productsCatálogo de productos (66+)SELECT público, INSERT/UPDATE/DELETE solo admin
profilesPerfiles de usuarios (mirror de auth.users)SELECT autenticados, UPDATE propio
ordersPedidos de clientesINSERT autenticados, SELECT propio + admin, UPDATE admin
pqrsPeticiones, Quejas, Reclamos, SugerenciasINSERT propio, SELECT propio + admin, UPDATE admin
reviewsReseñas y calificacionesSELECT público, INSERT propio, DELETE admin
couponsCupones de descuentoSELECT público, INSERT/UPDATE/DELETE solo admin
site_configConfiguración del sitio (banners, redes sociales)SELECT público, UPDATE solo admin

RPCs de seguridad (SECURITY DEFINER):

  • is_admin() — Verifica si el usuario actual es admin
  • list_users() — Lista todos los usuarios (solo admin)
  • admin_delete_user(uuid) — Elimina un usuario (solo admin)
  • admin_update_password(uuid, text) — Cambia contraseña de usuario (solo admin)
  • handle_new_user() — Trigger que crea perfil automáticamente al registrarse

6.9 Notificaciones Toast

Las notificaciones tipo toast aparecen en la esquina inferior derecha. Soportan tres tipos: success, error, info. Se eliminan automáticamente después de 3 segundos con fallback de eliminación por setTimeout.

6.10 Cupones de descuento

Sistema completo de cupones gestionado desde la pestaña "Cupones" del panel admin. Permite crear códigos de descuento con las siguientes opciones:

  • Código único — Texto alfanumérico (se convierte a mayúsculas automáticamente)
  • Tipo de descuento — Porcentaje (%) o monto fijo ($COP)
  • Compra mínima — Monto mínimo del carrito para aplicar el cupón
  • Límite de usos — Máximo de veces que se puede usar (0 = ilimitado)
  • Fecha de vencimiento — Expiración automática
  • Estado — Activo/inactivo con conteo de usos actual

Persistencia en Supabase (tabla coupons). Integrado con el formulario de pedido del carrito.

6.11 Analíticas avanzadas

Dashboard de analíticas en la pestaña "Analíticas" del panel admin con:

  • KPIs — Ingresos totales, total pedidos, ticket promedio, pedidos pendientes, completados, productos activos
  • Gráfico de ventas — Ventas por día con barras CSS responsivas
  • Top 10 productos — Los productos más vendidos con cantidad
  • Ventas por categoría — Distribución con porcentajes
  • Pedidos por estado — Pendiente, procesando, enviado, entregado, cancelado
  • Actividad reciente — Últimos 15 pedidos con estado
  • Filtro por fechas — 7, 30, 90, 365 días o todo el historial

6.12 PWA (Progressive Web App)

El sitio es instalable como aplicación gracias al Service Worker (sw.js) y el manifiesto (manifest.json). Estrategia de caché: cache-first para assets estáticos (CSS, JS, imágenes), network-first para páginas HTML. Los assets se actualizan en background (stale-while-revalidate). Las peticiones cross-origin (Supabase, CDN) se excluyen del caché.
